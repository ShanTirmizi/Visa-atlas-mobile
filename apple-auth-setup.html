03Build a fresh dev client.

Troubleshooting.

Things that commonly go wrong with the native flow.

"NATIVE MODULE CANNOT BE NULL" / "EXPO-APPLE-AUTHENTICATION COULD NOT BE FOUND" You're running the old dev client without the new native module. You must ship a new build (step 03) — Metro hot-reload doesn't add native modules to an existing binary.
"APPLE IDENTITY TOKEN MISSING SUBJECT" / TOKEN VERIFICATION FAILS The bundle ID in convex/AppleNative.ts (BUNDLE_ID constant) doesn't match the iOS app's bundle. Confirm both say com.tirmazilabs.visaatlas. If you ever rename the bundle, update the constant + redeploy Convex.
SHEET APPEARS BUT THEN ERRORS WITH "ERR_REQUEST_NOT_HANDLED" Sign in with Apple isn't enabled on your App ID's capabilities. Re-check section 02 — the Sign In with Apple capability must be ticked and saved.
SHEET DOESN'T APPEAR AT ALL The usesAppleSignIn: true entitlement isn't in the binary. Double-check app.json has it under ios, then build again. EAS prebuild reads this on each build.
NO MORE JWT RENEWAL Unlike the Web OAuth approach, the native flow has nothing to expire. Apple's public keys auto-rotate; jose handles the JWKS refresh; no calendar reminder needed.
